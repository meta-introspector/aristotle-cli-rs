# anchor-verify — Policy Verification Engine

Hard quality gate that scans local modifications to block commits if architectural boundaries are broken. Deploys a Git pre-commit hook and provides a standalone verification command for CI pipelines and manual audits.

## Parameters

| Parameter | Values | Default |
|-----------|--------|---------|
| `target_scope` | `staged`, `all` | `staged` |

- `staged` — verify only files staged in Git (`git diff --cached`)
- `all` — verify every tracked source file in the repository

---

## Preconditions

Before running `anchor-verify`, `anchor-init` must have been run in this repository. Verify:

```bash
test -f .anchor/config.json || { echo "ERROR: .anchor/config.json not found. Run anchor-init first."; exit 1; }
test -f .anchor/anchor-manifest.json || { echo "ERROR: .anchor/anchor-manifest.json not found. Run anchor-init first."; exit 1; }
```

If either is missing, stop and tell the user to run `anchor-init` first.

## Execution Steps

### Step 1 — Deploy the Pre-commit Hook (First Run Only)

Check if a pre-commit hook already exists:

```bash
HOOK_PATH=".git/hooks/pre-commit"
```

If `$HOOK_PATH` already exists, check if it was generated by anchor. Look for the marker:

```bash
grep -q "Generated by anchor-init" "$HOOK_PATH"
```

If the marker is NOT present, append the anchor hook logic to the existing hook (do not overwrite it). If it IS present, skip — already installed.

If no hook exists, create it with the full anchor verification logic.

#### Hook Content

```bash
#!/usr/bin/env bash
# Generated by anchor-init execution — anchor-verify pre-commit hook
# Marker: ANCHOR_PRECOMMIT_V1

set -euo pipefail

# Determine supported extensions from .anchor/config.json
# Fall back to a default set if config missing
if [ -f .anchor/config.json ]; then
    LANGUAGE=$(python3 -c "
import json
cfg = json.load(open('.anchor/config.json'))
# Derive languages from config if present, else default
print('ts,tsx,js,jsx,rs,py,lean')
" 2>/dev/null || echo "ts,tsx,js,jsx,rs,py,lean")
else
    LANGUAGE="ts,tsx,js,jsx,rs,py,lean"
fi

# Convert comma-separated to grep -E pattern
EXT_PATTERN=$(echo "$LANGUAGE" | tr ',' '|')
EXT_PATTERN="\.($EXT_PATTERN)$"

STAGED_FILES=$(git diff --cached --name-only --diff-filter=ACM | grep -E "$EXT_PATTERN" || true)

if [ -z "$STAGED_FILES" ]; then
    exit 0
fi

echo "🛡️  Executing anchor-verify..."

VIOLATIONS=0
for FILE in $STAGED_FILES; do
    # Ignore configuration files and test files (from config exclude rules)
    if echo "$FILE" | grep -qE '(test|spec|__tests__)'; then
        continue
    fi

    if ! grep -q "@anchor" "$FILE"; then
        echo "❌ POLICY VIOLATION: '$FILE' is missing an @anchor token."
        echo "   Agentic engines cannot map your execution pathway without structural boundaries."
        VIOLATIONS=$((VIOLATIONS + 1))
    fi
done

if [ $VIOLATIONS -gt 0 ]; then
    echo ""
    echo "🚫 Commit blocked: $VIOLATIONS file(s) missing @anchor tokens."
    echo "   Add an @anchor block to each file, then stage and commit again."
    echo ""
    echo "   Format for this project:"
    echo "     typescript/js:  /** @anchor [id] */  /** @tags [tags] */"
    echo "     rust:           // @anchor [id]      // @tags [tags]"
    echo "     python:         # @anchor [id]       # @tags [tags]"
    echo "     lean:           /-- @anchor [id]         @tags [tags] -/"
    exit 1
fi

# Force synchronization of the manifest map
if [ -f .anchor/generate_manifest.py ]; then
    python3 .anchor/generate_manifest.py > /dev/null 2>&1
    if [ -f .anchor/anchor-manifest.json ]; then
        git add .anchor/anchor-manifest.json 2>/dev/null || true
    fi
fi

echo "✅ Anchor manifest verification complete. State synchronized."
```

Make it executable:

```bash
chmod +x .git/hooks/pre-commit
```

### Step 2 — Run Verification

Depending on `target_scope`:

#### Scope: `staged` (default)

Scan files staged for commit:

```bash
STAGED_FILES=$(git diff --cached --name-only --diff-filter=ACM | grep -E '\.(ts|tsx|js|jsx|rs|py|lean)$' || true)

if [ -z "$STAGED_FILES" ]; then
    echo "No staged source files to verify."
    exit 0
fi
```

#### Scope: `all`

Scan every tracked source file:

```bash
STAGED_FILES=$(git ls-files | grep -E '\.(ts|tsx|js|jsx|rs|py|lean)$' || true)

if [ -z "$STAGED_FILES" ]; then
    echo "No source files found in repository."
    exit 0
fi
```

### Step 3 — Validate Each File

For each file in the scan list:

1. Skip files matching exclude patterns from `.anchor/config.json` (tests, specs, node_modules, etc.)
2. Check for the presence of `@anchor` token
3. If missing, flag as a violation
4. If present, validate the format:
   - Has a valid anchor ID (alphanumeric with `:` and `-`)
   - Has a `@tags` line (warning if missing, not a hard block)
   - Tags are from the allowed prefix set

### Step 4 — Manifest Sync

After validation passes, regenerate the manifest to reflect any changes:

```bash
if [ -f .anchor/generate_manifest.py ]; then
    python3 .anchor/generate_manifest.py
fi
```

### Step 5 — Report Results

Print a summary report:

```
anchor-verify — {$target_scope} scope

  Files scanned:   {N}
  Files passed:    {P}
  Files with warnings: {W}
  Files blocked:   {B}

  Manifest:        .anchor/anchor-manifest.json → {X} anchors ({Y} locations)
  Pre-commit hook: {installed/updated/already present}
```

#### If violations exist:

```
❌ VIOLATIONS DETECTED

  src/services/user.ts
    Missing @anchor token. Add:
      /** @anchor domain:user-service
       * @tags domain, api
       */

  src/infra/logger.py
    Missing @anchor token. Add:
      # @anchor infra:logger
      # @tags infra

  Commit blocked. Fix {B} file(s) and try again.
```

#### If all clear:

```
✅ All {N} files pass anchor verification.
   Manifest synchronized. Safe to commit.
```

## CI/CD Integration

To run anchor-verify in CI (e.g., GitHub Actions), add this step:

```yaml
- name: Anchor Verify
  run: |
    python3 .anchor/generate_manifest.py
    # Check all tracked files (no git diff --cached in CI)
    VIOLATIONS=0
    for f in $(git ls-files | grep -E '\.(ts|tsx|js|jsx|rs|py|lean)$'); do
      if echo "$f" | grep -qE '(test|spec|__tests__)'; then continue; fi
      if ! grep -q "@anchor" "$f"; then
        echo "VIOLATION: $f missing @anchor"
        VIOLATIONS=$((VIOLATIONS + 1))
      fi
    done
    if [ $VIOLATIONS -gt 0 ]; then
      echo "$VIOLATIONS files missing @anchor tokens"
      exit 1
    fi
    echo "All files have @anchor tokens."
```

## Manual Uninstall

To remove the pre-commit hook:

```bash
rm .git/hooks/pre-commit
```

Note: This removes the entire hook. If you had other pre-commit logic appended, restore it from backup.
